## tmux Setup

This script creates a project-based tmux session with ready-to-use windows.

![Tmux layout](/blog/omarchy-lazyvim-workflow/tmux-setup.png)

```bash
#!/bin/bash

PROJECT_DIR="$(pwd)"
SESSION="$(basename "$PROJECT_DIR")"

tmux has-session -t "$SESSION" 2>/dev/null

if [ $? != 0 ]; then
    tmux new-session -d -s "$SESSION" -n nvim -c "$PROJECT_DIR"
    tmux send-keys -t "$SESSION:nvim" "nvim ." C-m

    tmux new-window -t "$SESSION" -n opencode -c "$PROJECT_DIR"
    tmux send-keys -t "$SESSION:opencode" "opencode ." C-m

    tmux new-window -t "$SESSION" -n zsh -c "$PROJECT_DIR"

    tmux new-window -t "$SESSION" -n pnpm-dev -c "$PROJECT_DIR"
fi

tmux select-window -t "$SESSION:nvim"
tmux attach -t "$SESSION"
```

## How to Set It Up

1. Save the script as:

   ```bash
   ~/dev-session.sh
   ```

2. Make it executable:

   ```bash
   chmod +x ~/dev-session.sh
   ```

3. Add alias to your shell config:

   ```bash
   alias dev="~/dev-session.sh"
   ```

4. Reload your shell:

   ```bash
   source ~/.zshrc
   ```

## How I Use It Daily

1. Open any project folder:

   ```bash
   cd your-project-folder
   ```

2. Run:

   ```bash
   dev
   ```

3. Your full environment is ready instantly:

   - `nvim` opens with your project
   - `opencode` is ready for project commands
   - `zsh` is available for general usage
   - `pnpm-dev` window is ready to run your dev server

4. Start your server when needed:

   ```bash
   pnpm dev
   ```

5. Navigate between windows using tmux:

   ```bash
   Ctrl + b, then window number
   ```

## What It Does

- Creates or reuses a tmux session named after the project folder
- Opens:
  - `nvim`
  - `opencode`
  - `zsh`
  - `pnpm-dev`
- Focuses on `nvim` and attaches to the session
